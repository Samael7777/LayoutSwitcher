Как создать список клавиатур. Извлечь KLID из HKL?

  1. Home
  2. Вопросы и ответы
  3. Как создать список клавиатур. Извлечь KLID из HKL?

Windows 10 / C ++ / Win32

Мне нужно составить список «установленных» клавиатур.

Компоненты моего «списка» должны содержать:

  • HKL для клавиатуры.
  • KLID для клавиатуры.
  • Значения значений реестра, полученные из:
  • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts
  • Пример: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\00000409
  • Значение: файл макета
  • Значение: текст макета

Единственный способ, которым я знаю перечисление клавиатур, - это использовать GetKeyboardLayoutList(), который возвращает список HKL.

Методика, которая работает для «стандартных» клавиатур (HKL 04090409, 04070407, ..).

TCHAR Buffer[50];
TCHAR Buffer2[50];
HKL LoadedKeyboardLayout;

// Hkl: 04090409
_stprintf_s(Buffer, (sizeof(Buffer)/sizeof(TCHAR)), _T("%08X"), (((UINT)Hkl >> 16) & 0xFFFF));
// Buffer: "0000409"
LoadedKeyboardLayout = LoadKeyboardLayout(Buffer, (UINT)0);
// It Loads

ActivateKeyboardLayout(LoadedKeyboardLayout, KLF_SETFORPROCESS);
// It Activates

GetKeyboardLayoutName(Buffer2); 
// Buffer2: "00000409"

// I can now fish the registry HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\00000409

Это НЕ работает, если «идентификатор устройства» НЕ 0000.

Из LoadKeyboardLayout() документации:

Имя загружаемого идентификатора языкового стандарта ввода. Это имя представляет собой строку, состоящую из шестнадцатеричного значения идентификатора языка (младшее слово) и идентификатора устройства (старшее слово). Например, американский английский имеет идентификатор языка 0x0409, поэтому основная раскладка американского английского называется 00000409. Варианты раскладки американского английского (например, раскладка Дворжака) называются 00010409, 00020409 и т. Д.

Если создать «настраиваемую» клавиатуру, я обнаружил, что невозможно получить «идентификатор устройства» с настраиваемой клавиатуры HKL.

Например:

  • Создайте пользовательскую клавиатуру US с помощью MKLCS.
  • Установите его.
  • «KLID» будет «A0000409».
  • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\a0000409
  • HKL будет 0xF0C00409 (как возвращено в списке HKL, созданном GetKeyboardLayoutList()).
  • Чтобы загрузить клавиатуру с помощью LoadKeyboardLayout(), требуется «A0000409».
  • Создать A0000409 из F0C00409 не представляется возможным.
  • Я также создал свою раскладку клавиатуры без MKLCS.
  • Я произвольно назвал его 00060409.
  • HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Keyboard Layouts\00060409
  • Это HKL - FFFE0409 (как возвращено в списке HKL, созданном GetKeyboardLayoutList()).
  • Невозможно создать 00060409 из FFFE0409.

Спасибо, Рита.

Кажется, что GetKeyboardLayoutList() создает список «загруженных» раскладок клавиатуры.

  • Раскладки клавиатуры, загружаемые системой, 0x0409409, ....
  • Раскладки клавиатуры устанавливаются через установку MKLCS.

Кажется, что любые клавиатуры, определенные в следующих разделах реестра, будут загружены при загрузке.

  • HKEY_CURRENT_USER\Keyboard Layout Обратите внимание на значения предварительной загрузки и замены.

(Обратите внимание, что в реестре разбросано множество других ключей раскладки клавиатуры, поэтому я не уверен, является ли HKEY_CURRENT_USER\Keyboard Layout фактическим ключом реестра, который определяет PreLoad - Substitutes.

  • HKEY_USERS\.DEFAULT\Keyboard Layout
  • HKEY_USERS\S-1-5-18 (Моя учетная запись)
  • HKEY_LOCAL_MACHINE\SYSTEM\Keyboard Layout )

Поэтому у меня нет сомнений в том, что пример Риты работает, потому что ее HKEY_CURRENT_USER\Keyboard Макет содержит следующие записи: Предварительная загрузка: 1 d0010804

Заменители: d0010804 a0000804

Возможно, записи были помещены установщиком MKLCS? Или, возможно, действие по добавлению клавиатуры через Настройки- ›Время и язык-› Щелкните Предпочитаемый язык- ›Параметры-› Добавить клавиатуру

Из ActivateKeyboardLayout() документации:

Идентификатор языка ввода должен быть загружен предыдущим вызовом функции LoadKeyboardLayout.

Поскольку a0000804 (HKL: F0C00804) фактически уже загружен, ActivateKeyboardLayout() работает.

Поскольку мой KLID: 00060409 не упоминается ни в одной из предзагрузок X \ Keyboard Layout и заменителей, я должен физически вызвать LoadKeyBoardLayout(L"00060409"), чтобы он появился в GetKeyboardList() HKL.

как получить KLID от HKL?

Кажется, что нет прямого готового метода для этого.

Обходной путь - получить список HKL, затем активировать макет с помощью HKL, а затем получить его KLID. Ниже приведен пример:

int cnt = GetKeyboardLayoutList(sizeof(hklArr)/sizeof(hklArr[0]), hklArr);
if(cnt > 0)
{
    printf("keyboard list: \n");
    for (UINT i = 0; i < cnt; i++)
    {
        printf("%x\n", (LONG_PTR)hklArr[i]);
        if (ActivateKeyboardLayout(hklArr[i], KLF_SETFORPROCESS))
        {
            WCHAR pName[KL_NAMELENGTH];
            if (GetKeyboardLayoutName(pName))
            {
                wprintf(L"layout name (KLID): %s\n", pName);
            }
        }
    }
}
